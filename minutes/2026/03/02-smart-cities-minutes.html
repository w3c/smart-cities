Collaboration with ETSI/FIWARE on NGSI-LD

Juanjo Hierro sharing screen.

<kaz> [slide 1]

juanjo: Martin Bauer could no be here today.
… I'll reference work we do in ETSI.
… FIWARE is a an Open Source initiative for smart solutions in different sectors, like facturies.
… It allows to create platforms.
… Our members are active in different standards bodies, including W3C and ETSI.
… An important part of our work is how to make different standards work together.

<kaz> [slide 2]

juanjo: This is based very much on Digital Twins, or also a model of the real world.
… Model constantly updated based on various sources.
… Digital Twins means not necessarily 3D representations.
… It is data management, involved in execution of actions.
… An approach based on a System of Systems.
… Can be used in an organization, such a city, a port...
… Work on standardize interchange of information, e.g., between a city and its port, or between the city and a logistics operator.

<kaz> [slide 3]

juanjo: Digital Twins and System of Systems approaches used for integration at multiple levels.

<kaz> [slide 4]

juanjo: A Digital Twin is a kind of replica of an asset.
… E.g., is sombody wants to make a claim about something, a street, e.g., then the claim will also be modeled as a digitial twin.
… We are going to add Actions to our model as well.
… Typically a Digital Twin also has location data.
… A Context is the state of all digital twins in the collection.
… A standard API is a first needed step.
… But the systems also need to be able to talk about the same concepts, with the same semantics, e.g., temperature.
… So need command data models for digital twins.

<kaz> [slide 5]

juanjo: NGSI-LD API developed in ETSI.
… NGSI-LD is a simple (but powerful) REST API.
… Entity types and attributes mapped to paths.
… HTTP verbs GET, PUT, PATCH, etc.
… API developed based on developers' experience.
… Includes e.g., geo-queries, e.g., all all entities in an area.
… Also notifications to subscribe to.
… Queries but also push of information.
… To support different visualizations, cen askfor results in the form of GeoJSON.
… But can also be in other formats, such as key-value pairs.
… Supports batch operations (operations on multiple objects at a time).
… Support temporal operations, such as getting time series.
… A Context Broker is an implementation.

<kaz> [slide 6]

juanjo: See https://github.com/smart-data-models

<kaz> [slide 7]

juanjo: Shows diagram of reference architecture.
… Showing it for Smart Cities.

juanjo: Examples: air quality, traffic level, sensors and devices to update attributes of the system.
… A Context Broker is a means to intregrate the information.
… Some information relevant for some processes.
… Decoupling the supplier of information form the consumer of that information.
… Consumer of data interacts with Context Broker, but need not care about how the info got there.
… Updates of data are a constant stream. Can now integrate AI to get insights from the incoming data.
… E.g., I may have a system to predict traffic.
… Can then add an attribute to the system with the predicted traffic.
… There may also be a 3D representation.
… Extension to data spaces: market for data.

<kaz> [slide 8]

juanjo: FIWARE used by more than 300 cities already.
